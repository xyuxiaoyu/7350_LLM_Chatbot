Create a personalized interactive AI chatbot

You can assign a role to your artificial intelligence. You can make it interesting, angry, or become an expert in food, technology, health or anything you want.

Things to remember before building an AI chatbot

In this tutorial, we will offer ordinary users some tips for creating personalized chatbots, including prompt engineering, code modification and gradio GUI interface design.

We hope it will be helpful for your project.

Platform freedom

You can build API-driven chatbots on any platform, whether it's a local platform or an online one.

For the local platform, we recommend that you can visit the "General Tutorial" to learn all the details.

For online platforms, we recommend that you use Google Colab. You can write and execute python code on this platform.

Easy to understand

This guide is prepared for ordinary users, and the examples in the instructions are explained very clearly.

Therefore, even if you have a rough understanding of computers, you can easily create your own artificial intelligence chatbot.

Prompt Engineering

Prompt engineering is the process of discovering prompts which reliably yield useful or desired results. Prompt is the input you provide, typically text, when interfacing with an AI model.

You can learn more through the Prompt Engineering Guide.

LLM Settings

  • Temperature: The lower the temperature, the more deterministic the results. Increasing temperature could lead to more randomness, which encourages more diverse or creative outputs.
  • Max Length: Specifying a max length helps you prevent long or irrelevant responses and control costs.
  • Frequency Penalty: The higher the frequency penalty, the less likely a word will appear again. This setting reduces the repetition of words in the model's response by giving tokens that appear more a higher penalty.

Elements of a Prompt

  • Instruction: A specific task or instruction you want the model to perform.
  • Context: External information or additional context that can steer the model to better responses.
  • Input Data: The input or question that we are interested to find a response for.
  • Output Indicator: The type or format of the output.

General Tips for Designing Prompts

  • Start Simple: You can start with simple prompts and keep adding more elements and context as you aim for better results.
  • The Instruction: You can use commands to instruct the model what you want to achieve.
  • Specificity: Be very specific about the instruction and task you want the model to perform. The more descriptive and detailed the prompt is, the better the results.
  • Avoid Impreciseness: It's often better to be specific and direct -- the more direct, the more effective the message gets across.
  • To do or not to do: Avoid saying what not to do but say what to do instead.

Code Modification

You only need to make a small change in the code and it will be personalized. The following is the method:

Open the chatbot Python file

Right-click on the chatbot Python file you created and select "Edit with xxx" (xxx means the code editor on your computer).

Modify the code

Find the line of code "message = [{...}]" and only modify the "content" part in this specific line of code.

Simply feed the information back to the AI and let it take on this role. Here, we hope the chatbot can become a food expert, so we convey the following message:

"You are an AI specialized in Food. Do not answer anything other than food-related queries."

Press "Ctrl + S" to save the file.

Run the Python file

Open the terminal, run the following command, and then click Enter. Please remember that the file paths of your computer will be different.

python "C:\Users\mearj\Desktop\app.py"

python3 "C:\Users\mearj\Desktop\app.py"

You will get a local and a public URL. Copy the local URL.

To stop the server, move to the terminal and press "Ctrl + C". If it doesn't work, press "Ctrl + C" again.

After every change you make to the python file, you must restart the server.

Get your AI chatbot

You will get a personalized AI chatbot that only answers questions related to food.

You can also create a doctoral artificial intelligence, an artificial intelligence that answers like Shakespeare, an artificial intelligence that speaks in Morse code, whatever you want.

Gradio GUI Interface Design

Gradio is an open-source Python package that allows you to quickly build a demo or web application for your machine learning model, API, or any arbitrary Python function. You can then share a link to your demo or web application in just a few seconds using Gradio's built-in sharing features.

Brief Introduction

  • Gradio is an open-source Python library for creating an interactive interface for machine learning models. Gradio is widely used in the fields of data science, education, research and software development, and is particularly suitable for rapid prototyping, model validation, demonstration and teaching.
  • The charm of Gradio lies in its ease of use. Without complex front-end knowledge, with just a few lines of code, you can transform any machine learning model into a beautiful and interactive interface. This not only makes model presentation simple, but also provides a window for people without a technical background to explore and understand AI.

Interface Elements: Input Component

  • Audio: Allows users to upload audio files or record directly.

    Parameters: source: Specifies the audio source (such as a microphone), type: Specifies the return type. Example: gr.Audio(source="microphone", type="filepath")

  • Checkbox: Provides checkboxes for Boolean value input.

    Parameters: label: The text label displayed next to the checkbox. Example: gr.Checkbox(label="Agreement Terms")

  • CheckboxGroup: Allows users to select multiple options from a set.

    Parameters: choices: String array, representing the options of the checkbox; label: Label text. Example: gr.CheckboxGroup(["Option 1", "Option 2"," Option 3"], label="Select your Interests")

  • ColorPicker: It is used to select colors and usually returns hexadecimal color codes.

    Parameters: default: The default color value. Example: gr.ColorPicker(default="#ff0000")

  • Dataframe: Allows users to upload CSV files or enter dataframes.

    Parameters: headers: Array of column headers; row_count: The number of rows initially displayed. Example: gr.Dataframe(headers=["Column 1", "Column 2"], row_count=5)

  • Dropdown: A drop-down menu from which users can select an option.

    Parameters: choices: String array, representing the options of the drop-down menu; label: Label text. Example: gr.Dropdown(["option 1", "option 2", "option 3"], label = "choose an option")

  • File: It is used to upload any file and supports multiple file formats.

    Parameters: file_count: The number of files allowed to be uploaded, such as "single" or "multiple", type: The returned data type, such as "file" or "auto". Example: gr.File(file_count="single", type="file")

  • Image: It is used for uploading pictures and supports multiple image formats.

    Parameters: type: image type, such as pil. Example: gr.Image(type='pil')

  • Number: Numeric input box, suitable for both integers and floating-point numbers.

    Parameters: default: default number, label: label text. Example: gr.Number(default=0, label="Input a number")

  • Radio: Radio button group, from which the user selects an option.

    Parameters: choices: String array, representing the options of radio buttons; label: Label text. Example: gr.Radio(["option 1", "option 2", "option 3"], label = "choose an option")

  • Slider: A sliding bar used to select values within a certain range.

    Parameters: minimum: minimum value, maximum: maximum value, step: Step size, label: Label text. Example: gr.Slider(minimum=0, maximum=10, step=1, label="Adjust value")

  • Textbox: A single-line text input box, suitable for short text.

    Parameters: default: default text, placeholder: placeholder text. Example: gr.Textbox(default="default text", placeholder="input text")

  • Textarea: A multi-line text input area, suitable for longer text input.

    Parameters: lines: Display the number of lines, placeholder: Placeholder text. Example: gr.Textarea(lines=4, placeholder="Enter long text")

  • Time: Used for inputting time.

    Parameters: label: Label text. Example: gr.Time(label="Select time")

  • Video: Video upload component, supporting multiple video formats.

    Parameters: label: Label text. Example: gr.Video(label="Upload video")

  • Data: The original bytes used for uploading binary data, such as images or audio.

    Parameters: type: Data type, such as "auto" for automatic inference. Example: gr.Data(type="auto", label="Upload data")

Interface Elements: Output Component

  • Audio: Play audio files.

    Parameters: type: specifies the output format. Example: gr.Audio(type="auto")

  • Carousel: Displays multiple outputs in a carousel mode, suitable for image sets or multiple data points.

    Parameters: item_type: sets the carousel item type. Example: gr.Carousel(item_type="image")

  • Dataframe: Display a Pandas DataFrame, which is suitable for tabular data.

    Parameters: type: specifies the type of the returned DataFrame. Example: gr.Dataframe(type="pandas")

  • Gallery: Display a series of images in the form of a gallery.

  • HTML: Displays HTML content, suitable for rich text or web page layouts.

  • Image: Display the image.

    Parameters: type: specifies the image format. Example: gr.Image(type="pil")

  • JSON: Display data in JSON format for easy viewing of structured data.

  • KeyValues: Display data in the form of key-value pairs.

  • Label: Display text label, suitable for simple text output.

  • Markdown: Supports text display in Markdown format.

  • Plot: Display a chart, such as one generated by matplotlib.

  • Text: Used for displaying text and suitable for longer output.

  • Video: Play video files.
